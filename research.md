### 6 разобранных портфолио (Dribbble / Behance)

#### 1. Pawel Gola — pawelgola.com
- **Стиль:** тёмный (#0a0a0a), электрический синий (#0057FF), grotesque 120–160px
- **Структура:** Hero → Selected Work → About → Services → Contact
- **Фишка:** Имя на весь экран. Hover на кейсах — цветной overlay с описанием без перехода
- **Взять:** числа-якоря (01 02 03), hover-preview вместо отдельных страниц

#### 2. Segun Akintunde — segunakintunde.com
- **Стиль:** оффвайт (#F5F2ED), тёплый оранжевый (#FF5C00), serif заголовки
- **Структура:** Hero → горизонтальный скролл Work → Process → Blog → Contact
- **Фишка:** горизонтальный скролл кейсов — нестандартное взаимодействие
- **Взять:** роль как animating tagline, serif + grotesque контраст

#### 3. Maxime Mougin — maximemougin.com
- **Стиль:** чёрный + белый, condensed grotesque, текст «прорывается» за поля
- **Структура:** Hero + marquee → Showreel видео → Projects → Studio → Contact
- **Фишка:** бегущая строка в hero задаёт ритм ещё до скролла
- **Взять:** marquee-строка, autoplay video reel как второй экран

#### 4. Fiona Vessel — fionavessel.com
- **Стиль:** пастель (молочный, sage green, пыльная роза), editorial serif italic
- **Структура:** Hero (фото вписано в типографику) → Featured Projects → Capabilities → Clients → Journal → Footer-CTA
- **Фишка:** фото встроено в типографическую сетку, не изолировано в кружок
- **Взять:** Capabilities через «/»: `Brand / Identity / Motion / UX`, footer-CTA как эмоциональное завершение

#### 5. Olivier Gillaizeau — oliviergillaizeau.com
- **Стиль:** тёмно-зелёный + кремовый, WebGL-фон реагирует на курсор, mono-шрифт в hero
- **Структура:** Hero (3D interactive) → Work → About (1 абзац + теги) → Contact (только email)
- **Фишка:** WebGL говорит «я умею это» без слов. Минимум текста — максимум показывает
- **Взять:** mono → обычный grotesque (переход «код → человек»), минималистичный contact

#### 6. Selin Cicek — selincicek.com
- **Стиль:** белый, угольный (#1A1A1A), один акцент — жёлтый (#FFD600) только для CTA
- **Структура:** Hero + availability badge → Case Studies → Skills → Testimonials → Resume → Contact
- **Фишка:** availability badge («Available for freelance») — мгновенно сообщает статус
- **Взять:** badge доступности, раздел Testimonials, прямая ссылка на скачивание CV

## ЧАСТЬ 2 — ИДЕАЛЬНАЯ СТРУКТУРА ЛЕНДИНГА

> Модель: AIDA + Trust Loop. Порядок блоков выстраивает путь от первого впечатления до действия.

```
┌──────────────────────────────────────────────┐
│  01. HERO              [обязательный]         │
│  02. SOCIAL PROOF BAR  [обязательный]         │
│  03. ABOUT             [обязательный]         │
│  04. SERVICES          [обязательный]         │
│  05. PORTFOLIO         [обязательный]         │
│  06. PROCESS           [рекомендуемый ★★★]   │
│  07. TESTIMONIALS      [обязательный]         │
│  08. FAQ               [рекомендуемый ★★★]   │
│  09. CONTACT / CTA     [обязательный]         │
│  10. FOOTER            [обязательный]         │
└──────────────────────────────────────────────┘
```

### Детали по каждому блоку

#### 01 · HERO
- **Цель:** удержать внимание за 3–5 секунд
- **Контент:** имя / бренд, ёмкий tagline-позиционирование, CTA-кнопка, визуальный якорь (фото / анимация)
- **Триггер:** Первое впечатление + Ясность — мозг принимает решение о доверии за 50 мс

#### 02 · SOCIAL PROOF BAR
- **Цель:** снять скептицизм сразу после Hero, пока внимание на пике
- **Контент:** логотипы клиентов (5–8), или счётчики (47 проектов / 6 лет), или одна крупная цитата
- **Триггер:** Социальное доказательство + Авторитет

#### 03 · ABOUT
- **Цель:** создать человеческое соединение — клиенты нанимают людей, не резюме
- **Контент:** живое фото, текст от первого лица (кто / как работаю / личная деталь), вторичный CTA
- **Триггер:** Симпатия (Liking) по Чалдини — люди работают с теми, кто нравится

#### 04 · SERVICES
- **Цель:** устранить неопределённость «а что ты предлагаешь?»
- **Контент:** 3–5 услуг с иконками, акцент на результате а не процессе
- **Триггер:** Ясность + снижение тревоги

#### 05 · PORTFOLIO / SELECTED WORKS
- **Цель:** доказать компетентность через работы
- **Контент:** 3–6 сильных кейсов с превью, задачей, результатом. CTA после секции
- **Триггер:** Демонстрация компетентности + Желание

#### 06 · PROCESS ★ (часто упускают)
- **Цель:** снять страх перед неизвестностью («а вдруг хаос и срывы сроков?»)
- **Контент:** 4–6 шагов сотрудничества (Discovery → Прототип → Дизайн → Запуск) с примерными сроками
- **Триггер:** Снижение риска + Предсказуемость

#### 07 · TESTIMONIALS
- **Цель:** передать слово клиентам — это сильнее любых собственных заявлений
- **Контент:** 3–5 отзывов с именем / должностью / фото / логотипом, КОНКРЕТНЫЕ цифры
- **Триггер:** Социальное доказательство + Идентификация

#### 08 · FAQ ★ (крайне недооценён)
- **Цель:** убрать последнее возражение у человека, уже готового написать
- **Контент:** 5–8 реальных вопросов (цена / сроки / удалённая работа / что если не понравится)
- **Триггер:** Устранение возражений + Доверие через прозрачность

#### 09 · CONTACT / CTA
- **Цель:** конвертировать интерес в конкретное действие
- **Контент:** активный заголовок («Начнём проект?»), форма 3–4 поля, мессенджеры, время ответа
- **Триггер:** Конкретность + Снижение трения

#### 10 · FOOTER
- **Цель:** навигационная страховка, финальные точки контакта
- **Контент:** логотип, соцсети, навигация, copyright
- **Триггер:** Завершённость

---

### Что упускают и теряют конверсию

1. **Process-блок** — клиент боится неизвестности, без него теряются на середине воронки
2. **FAQ** — финальный барьер перед отправкой, 80% портфолио его игнорируют
3. **Конкретные метрики** в отзывах (+34% конверсии) вместо «всё понравилось»
4. **Время ответа** в контактах — «отвечу за 24 часа» повышает кол-во форм
5. **Secondary CTA** после секции работ — пиковый момент интереса, который большинство теряет

## ЧАСТЬ 3 — ТРЕНДЫ 2025–2026

### Топ-10 трендов (применимые для портфолио)

| # | Тренд | Описание | Сложность | Применимо |
|---|-------|----------|-----------|-----------|
| 1 | **Bento Grid Layout** | Асимметричная CSS Grid сетка карточек разного размера (стиль Apple) | Низкая | ✅ Да |
| 2 | **Scroll-Driven Animations** | CSS анимации по прогрессу скролла без JS — `animation-timeline: view()` | Средняя | ✅ Да |
| 3 | **Kinetic Typography** | Бегущие строки (marquee), буквы по одной, заголовки с анимацией | Средняя | ✅ Да |
| 4 | **Glassmorphism 2.0** | Многослойное размытие + цветные блики, frosted glass на тёмном фоне | Низкая | ✅ Да |
| 5 | **Bold Exaggerated Typography** | Драматически крупный заголовок + крошечный подтекст, резкий контраст размеров | Низкая | ✅ Да |
| 6 | **Fluid Typography (clamp)** | Шрифты и отступы масштабируются плавно без медиазапросов через `clamp()` | Низкая | ✅ Да |
| 7 | **Custom Cursor с инерцией** | Кастомный курсор с lerp-инерцией, меняет форму на hover-элементах | Средняя | ✅ Да |

| 9 | **Archival Index Style** | Редакционные сетки, приглушённая палитра, footnotes — «цифровой архив» | Низкая | ✅ Да |
| 10 | **Micro-interactions** | Кнопка «вжимается», стрелка поворачивается, hover-состояния с clip-path | Средняя | ✅ Да |

### Детали ключевых трендов

#### Bento Grid — секция портфолио
```css
.bento-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: 200px;
  gap: 16px;
}
.card--wide { grid-column: span 2; }
.card--tall { grid-row: span 2; }
```

#### Scroll-Driven Animations (нативный CSS, без GSAP)
```css
@keyframes fade-in {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}
.section-title {
  animation: fade-in linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}
```

#### Marquee (бегущая строка)
```css
.marquee-track {
  display: flex;
  animation: marquee 20s linear infinite;
  white-space: nowrap;
}
@keyframes marquee {
  from { transform: translateX(0); }
  to   { transform: translateX(-50%); }
}
```

#### Custom Cursor с инерцией
```javascript
let cx = 0, cy = 0, mx = 0, my = 0;
document.addEventListener('mousemove', e => { mx = e.clientX; my = e.clientY; });
function animate() {
  cx += (mx - cx) * 0.1;
  cy += (my - cy) * 0.1;
  cursor.style.transform = `translate(${cx}px, ${cy}px)`;
  requestAnimationFrame(animate);
}
animate();
```
