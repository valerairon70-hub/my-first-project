# Правила работы с проектом

## Технические требования

- **Один HTML-файл** — каждая страница полностью самодостаточна: CSS и JS внутри файла, без внешних библиотек, без CDN, без импортов
- **Иконки** — только SVG, встроенные прямо в HTML
- **Изображения** — base64, встроены в HTML-файл
- **Хостинг** — GitHub Pages (`valerairon70-hub.github.io/my-first-project/`)

## Дизайн

Единый стиль для всех страниц проекта (NeuroTracker AI + визитка):

- **Фон:** тёмный — `#0a0a0a` или близкие оттенки
- **Текст:** светлый — `#e8e8e8` и вариации
- **Акцентный цвет:** оранжево-золотой — `#f0a500`, `#e85d04`, `#ff6b35`
- **Карточки навигации/структуры:** glassmorphism — `rgba(255,255,255,0.04)` + `backdrop-filter: blur()`
- **Карточки с длинным текстом** (расшифровки, отзывы, FAQ): светлый фон `rgba(255,255,255,0.08)` + тёмный текст для лучшей читаемости
- **Кнопки:** градиент от золота к оранжу, glow-эффект при hover
- **Шрифт:** системный стек — `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- **Размер основного текста:** минимум 16px — аудитория 35–55 лет, телефон

> Почему этот стиль: тёмный фон сигнализирует «серьёзный AI-инструмент», золото/оранж добавляет тепло и доверие — это важно для ниши здоровья. Фиолетово-cyan стиль (как Linear, Vercel) не подходит: он создан для технарей, а наша аудитория — женщины 35–55 лет, не технари.

## Адаптивность

- Mobile-first — вёрстка начинается с мобильного, затем расширяется
- Минимальная ширина: 320px
- Размеры шрифтов через `clamp()` — плавно масштабируются без медиазапросов
- Все кнопки и ссылки — достаточно крупные для нажатия пальцем (min 44px)
- Проверять на телефоне обязательно

## Стиль текста

- Простой и дружеский — как разговор с живым человеком
- Без канцелярита: не «осуществляется», а «делается»; не «в рамках данного проекта», а «здесь»
- Конкретика вместо общих слов: цифры, примеры, сценарии из жизни
- Никакой MLM-риторики: не «прорыв», не «уникальный», не «трансформируй»

## Рабочий процесс

- **Перед созданием нового файла** — сначала описать, что планируется сделать, и дождаться подтверждения
- **Перед большими изменениями** в существующем файле — коротко объяснить что меняется и почему
- **Фото пользователя** уже встроено в `vizitka.html` в base64 — брать оттуда при необходимости
- Не добавлять функционал, о котором не просили
- Не рефакторить код, если задача этого не требует
